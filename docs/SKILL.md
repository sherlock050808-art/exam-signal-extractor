# 开卷闭卷考试助手

当用户需要准备考试、整理考点、生成考场资料时，使用本 Skill。核心目标不是总结知识，而是**从海量教学信息中提取对考试最有预测价值的信号，并生成可执行的考试地图**。

> **关键认知**：录音决定考试范围，PPT/课件填充知识内容。不要把二者等权处理。

## 一、触发条件

以下任一情况即触发：

- 用户提到“考试复习”“考前冲刺”“开卷考试”“闭卷考试”
- 用户提供课堂录音、课件 PDF、随堂练习、考点文件
- 用户要求整理考点、生成小抄、生成考场资料
- 用户要求做名词解释、简答题、综合分析题整理

## 二、触发后第一步：确认考试形式

**必须在整理任何内容之前确认考试形式。** 闭卷与开卷的资料形态、输出结构、记忆策略完全不同，先确认形式再决定后续流程。

若用户已明确说明，直接按对应策略执行；若未说明，必须主动询问：

> “这场考试是开卷还是闭卷？两种形式的复习资料结构差别很大，我先确认一下。”

| 确认结果 | 立即决定的事项 |
|---------|---------------|
| 闭卷 | 采用“题型分布前置 + 分层知识点 + 采分点背诵”结构 |
| 开卷 | 采用“考试地图 + 完整答案 + 强索引”结构 |
| 待定 | 先生成通用考试地图，待确认后再派生具体资料 |

## 三、核心原则

1. **信号 > 内容**：老师关于考试的话比知识本身更有预测力。录音里真正有价值的信号占比可能不到 10%，但对预测考题的价值可能超过 90%。
2. **录音决定范围，PPT/课件填充内容**：
   - 录音是考试范围信号源（老师在告诉你考什么）。
   - PPT/课件是知识库（老师在告诉你怎么考、考哪些具体内容）。
   - 不要把二者等权处理，否则产出会颠倒为 70% 课件、30% 录音。
3. **同一套七层框架，不同权重配置**：
   - 开卷和闭卷共用同一张考试地图（🟢 必考层、🟡 高概率层、🟠 可能层、🔴 排除层、⚡ 陷阱区、🔗 融合区、❓ 待确认）。
   - 差异不是两套逻辑，而是各层的**覆盖深度**和**投入权重**不同：闭卷向 🟢🟡 层倾斜，但仍需评估 🟠 层；开卷对 🟠 层也要覆盖，但不必全部深度记忆。
4. **排除层 = 必考层**：明确“不考”的内容同样要记录，节省复习精力。
5. **知识密度 ≠ 考试价值**：老师没强调但课件大标题的内容，也可能是考点。
6. **意图在交际动作层面，不在句子层面**：单个句子往往无法独立表达意图，必须先把句子合并为意图单元，再对单元分类。
7. **课件对照是必要步骤**：录音没提但课件为标题/独立条目的内容，要升级为“可能层”。

## 四、开卷 vs 闭卷策略差异

二者使用同一套七层考试地图，差异体现在**覆盖阈值**和**投入权重**。

| 七层 | 开卷考试（覆盖阈值高） | 闭卷考试（向高层倾斜） |
|------|----------------------|----------------------|
| 🟢 必考层 | 完整答案 + 索引定位 + 快速照抄 | 完整记忆 + 能默写 + 快速调用 |
| 🟡 高概率层 | 完整答案 + 索引定位 | 重点记忆采分点 + 能展开 |
| 🟠 可能层 | 名词解释覆盖 / 简答采分点覆盖，便于查找 | 了解即可，识别关键词，不逐题准备 |
| 🔴 排除层 | 明确标记，不写入考场资料 | 明确标记，不投入复习 |
| ⚡ 陷阱区 | 整理易混点 + 区分方式，便于查找 | 重点记忆区分方式 |
| 🔗 融合区 | 准备完整综合分析答案 | 准备通用答题框架 + 常见组合 |
| ❓ 待确认 | 考前尽量确认，确认后升入对应层 | 考前尽量确认，若无法确认则按低层处理 |

| 支持维度 | 开卷考试 | 闭卷考试 |
|---------|---------|---------|
| 资料厚度 | 厚，但必须可检索 | 薄，按七层分层清晰 |
| 索引要求 | 必须有章节/题型/关键词索引 | 可按概率分层排列 |
| 记忆强度 | 以识别和定位为主 | 以默写和调用为主 |
| 时间分配 | 各层都要整理，但 🟢🟡 写完整答案 | 🟢🟡 占 70% 以上精力，🟠 占 20%，其余 10% |
| 前置页 | 考试地图 + 快速索引 | **题型分布**（默认）或目录（按用户偏好） |

**闭卷考试特别说明**：闭卷版资料建议以“题型分布”作为前置页，让学生拿到资料后第一件事就是知道每种题型会考什么。但此结构并非强制，需询问用户是否接受；若用户希望保留目录，则改为目录 + 文内按题型标注。

## 五、六步工作流

```
原始转录文本（用户提供）
    │
    ├─→ [步骤1 硬过滤] 删除确定性噪声（课堂动作、元数据、纯语气词）
    │    削减 ~30% token
    │
    ├─→ [步骤2 话题切分] 第一层：段落级，按主题切成 15-20 段
    │
    ├─→ [步骤3 意图单元合并+粗分类] 第二层：意图段级
    │      把连续句子合并为意图单元，判考试信号 / 知识内容 / 噪声 / 人工判定
    │      → 考试信号进入步骤 4
    │      → 知识内容进入课件对照通道
    │
    ├─→ [步骤4 精细提取] 第三层：句子级，对考试信号做结构化提取
    │
    ├─→ [步骤5 课件对照验证] 把课件标题级内容升入可能层
    │
    └─→ [步骤6 输出考试地图] 按七层结构重组
```

### 三层切片精度

| | 第一层 | 第二层 | 第三层 |
|--|--------|--------|--------|
| 做什么 | 话题在哪切换 | 老师在干什么 | 具体考点是什么 |
| 处理单元 | 全文 | 30-50 句的段落 | 3-10 句的意图单元 |
| 判断类型 | 边界检测 | 意图粗分类 | 结构化精提取 |
| 用什么 | 规则 + embedding | 小模型批处理 | 大模型精提取 |
| 成本 | 几乎为零 | 低 | 高（但只覆盖约 20-40% 内容） |

### 步骤 1：硬过滤

删除以下确定性噪声：

- 录音元数据
- 课堂动作指令（“翻到第 X 页”“抬头看黑板”）
- 纯语气词（“好。”“对吧？”“嗯。”）
- 独立过渡引导语（≤10 字）
- 与课程无关的闲聊

**注意**：承载考试信号的过渡句不能删，例如“接下来这是我们课程的重点”。硬过滤后抽查 10 条，确认没有误删。

### 步骤 2：话题切分

按显性切换词或 embedding 相似度骤降处切分，产出 15-20 个主题段落。

常见切换词：

- “接下来我们讲……”
- “第二个问题……”
- “最后一个……”
- “另外还有……”

### 步骤 3：意图单元合并 + 粗分类

把连续句子合并为完成一个交际动作的“意图单元”，粗分为四类：

| 大类 | 定义 | 下一步 |
|------|------|--------|
| 考试信号 | 传递考试相关信息 | → 步骤 4 |
| 知识内容 | 纯知识讲解，未涉考试 | → 课件对照通道 |
| 噪声 | 闲聊/管理/无关 | → 丢弃 |
| 人工判定 | 置信度低 | → 人判后入 few-shot 库 |

常见交际模式：

| 模式 | 结构 | 典型意图 |
|------|------|----------|
| A | [设问] → [核心回答] → [补充举例] | 必考点名 |
| B | [提醒注意] → [陈述规则] → [举例] | 答题策略 |
| C | [铺垫] → [下结论/划范围] | 范围信号 |
| D | [闲聊] → [展开] → [收尾] | 噪声 |

### 步骤 4：精细提取

对考试信号单元提取结构化信息：

```json
{
  "知识点": "具体考点名称",
  "关联题型": "名词解释 | 填空 | 简答 | 综合分析",
  "确定性": "必考 | 高概率 | 可能 | 排除",
  "卷面": "A卷 | B卷 | 通用",
  "答题要点": "老师提到的具体答题要求",
  "关联知识点": ["跨章节关联的其他考点"],
  "原文定位": "句子索引区间"
}
```

### 步骤 5：课件对照验证

**这是开卷考试最关键的一步。**

对课件中的内容按以下规则判定：

| 课件形式 | 判定 |
|----------|------|
| 标题/加粗/独立条目 | 考点或正范围信号 |
| 正文/注释/脚注 | 知识复述 |
| 案例/配图/拓展阅读 | 无效或低价值 |

把所有课件标题级概念加入“可能层”，并写一句话定义。

### 步骤 6：输出考试地图

按以下七层结构输出，不按录音顺序：

| 层次 | 内容 |
|------|------|
| 🟢 必考层 | 知识点 + 题型 + 卷面 + 答题要点 |
| 🟡 高概率层 | 同上，确定性稍低 |
| 🟠 可能层 | 知识点 + 可能题型（开卷考试要重点扩充） |
| 🔴 排除层 | 知识点 + 不考原因 + 原文依据 |
| ⚡ 陷阱区 | 知识点 + 常见错误 + 区分方式 |
| 🔗 融合区 | 知识点组合 + 可能的综合题形式 |
| ❓ 待确认 | 原文 + 分类器猜测 + 阻塞原因 |

## 六、输出格式

输出结构应根据考试形式选择：

- **闭卷考试**：默认采用“题型分布前置 + 分层知识点 + 公式/代码速查”结构。
- **开卷考试**：默认采用“考试地图 + 完整答案 + 强索引”结构。

> **关于题型分布页**：题型分布页把知识点按“选择/判断、简答、分析、计算、编程”等题型归类。实践表明，闭卷复习时这比传统目录更实用。但是否采用应询问用户，因为有些用户可能希望保留目录或按章节索引。询问方式：
>
> “闭卷版我打算把传统目录换成‘题型分布’页，按选择/判断、简答、分析、计算、编程列出考点，方便你直接按题型突击。你想要这种结构，还是保留按章节的目录？”

### 1. 名词解释清单

按章节或字母排序，每个术语一行：

```
术语 —— 一句话定义（含核心关键词）
```

开卷考试必须覆盖课件中所有标题级术语。

### 2. 简答题

格式：

```
采分点 1
采分点 2
采分点 3 —— 完整答案：把上述采分点串成一段可直接照抄的话
```

### 3. 综合分析题

按“完整可抄答案”准备，不是模板。包含：

- 完整流程（每个环节）
- 具体装备/工艺名称
- 智能化特征
- 总结句

### 4. 索引

开卷资料必须包含：

- 按章节索引
- 按题型索引
- 关键词首字母索引

## 七、质量检查清单

- [ ] 是否已确认考试形式（开卷/闭卷）？
- [ ] 输出结构是否与考试形式匹配？（闭卷：题型分布前置；开卷：完整索引）
- [ ] 硬过滤是否误删了携带考试信号的句子？（抽查 10 条）
- [ ] 话题切分是否在合理位置？
- [ ] 负范围信号是否全部被捕捉？
- [ ] 课件中所有标题级概念是否进入“可能层”？
- [ ] 名词解释是否覆盖课件全部标题术语？（开卷考试关键）
- [ ] 简答题是否采分点 + 完整答案都有？
- [ ] 综合分析题是否是可直接照抄的完整答案？
- [ ] 是否有按章节/题型/关键词的索引？
- [ ] 考后是否填写复盘清单？

## 八、常见陷阱

1. **不要默认考试形式**：用户没有明确说开卷或闭卷时，必须先询问。两种形式的资料结构完全不同，默认错误会导致一轮返工。
2. **过渡句 ≠ 噪声**：“接下来这是我们课程的重点”是范围划定，不能删。硬过滤规则要对 ≤10 字的过渡句设长度门槛，带实质内容的不能删。
3. **负信号比正信号更容易漏**：“了解一下就行”“不要求掌握”语速快、音量低，ASR 可能模糊，需要专门检索。
4. **设问句不要单独判**：“这个地方会考什么？”单独看没有任何信号，必须和后续回答合并为一个意图单元才能正确分类。
5. **同一知识点多次出现 ≠ 强调**：闲聊、课堂管理、划重点三种语境中都可能出现同一个术语，只有划重点语境中出现才是考试信号。
6. **知识内容不是自动等于低价值**：课件大标题要升级为考点。
7. **正信号内部有强度差异**：第一人称直接确认 > 一般提醒；唯一性信号 > 包含性信号。
8. **输出完整 ≠ 信号准确**：最终考试地图要做两层验证：对照课件验证知识内容，对照常识验证信号判断是否合理。完整漂亮的文档和高命中率的考试地图是两件事。
