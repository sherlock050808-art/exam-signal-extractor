# 从课堂录音中提取考试信号：完整方法论

## 一、问题的重新定义

这套方法论要解决的不是"怎么做复习资料"，而是一个更根本的问题：**如何从海量教学信息中提取对考试最有预测价值的信号。**

两者的差距比看起来要大。常见的 Agent 流程是把录音和 PPT 当作两个同等重要的信息源，合并总结，生成一份结构完整的笔记。这个流程的问题在于，它误解了两种材料各自的角色：

- **录音 ≈ 考试范围**（老师在告诉你考什么）
- **PPT ≈ 知识库**（老师在告诉你怎么考）

**录音决定范围，PPT 填充内容。** 如果把二者等权处理，加上课件动辄 100-200 页，最终产出往往是 70% 来自课件、30% 来自录音——恰好颠倒了应有的比重。

更根本的问题是：**录音不是知识载体。** 录音里真正有价值的信号，不是老师讲了什么知识，而是老师对知识的态度、对考试的暗示、对学生错误的观察。这些信号在原始文本里占比可能不到 10%，但对预测考题的价值可能超过 90%。

因此，整个系统的目标不是：

```
录音 + PPT → 总结
```

而是：

```
录音 → 语句分类 → 教学意图提取 → 出题模型重建 → 考试地图
```

产出不是复习笔记，是**考试地图**——标记必考、大概率、可能、不考、陷阱区、跨章节融合区。

## 二、分类体系：先粗后细，两层递进

### 第一层：粗分类（3 类 + 兜底桶）

第一层不做精细判断，只回答一个问题：**这段话和考试有没有关系？**

| 大类 | 定义 | 分流方向 |
|------|------|----------|
| 考试信号 | 老师在传递关于考试的任何信息 | 进入第二层细分类 |
| 知识内容 | 老师在讲知识本身，未涉及考试 | 进入课件对照验证通道 |
| 噪声 | 与教学无关的一切 | 丢弃 |
| ⚡ 人工判定 | 置信度低于阈值的模糊边界情况 | 标记，交由人工判断 |

**人工判定桶的设计原则**：

- 分类器对任何置信度低于阈值（建议 0.8）的样本，不强判，不丢弃，统一归入人工判定桶。
- 人工判完之后，该样本自动成为 few-shot 例句，提升后续分类准确率。
- 这个桶同时是**质量兜底**和**持续改进机制**。

第一层之所以只做粗筛，是因为精度不是这一层的目标——省成本才是。假设全文 70% 是噪声和知识内容，30% 是考试信号，那么你不需要对 100% 的句子做精细的 7 选 1，只需要对 30% 的句子做后续的细分类。第一层的 token 消耗极低，小模型加 few-shot 即可胜任。

### 第二层：细分类（仅对"考试信号"展开，7 类）

第二层只对通过第一层筛选的"考试信号"单元执行，回答：**老师在传递哪一类考试信号？**

| 意图类别 | 定义 | 典型触发模式 | 考试价值 |
|----------|------|-------------|:--------:|
| 题型号令 | 明确说明题型、题量、分值、考试结构 | "有名词解释、填空、简答、综合分析" | 最高 |
| 必考点名 | 明确指认某知识点会考、以什么形式考 | "这个一定会考名词解释""我肯定考" | 最高 |
| 正范围信号 | 暗示/明示需要重点关注 | "注意一下""这个地方小心""要掌握" | 高 |
| 负范围信号 | 明确/暗示不考或只需了解 | "了解一下就行""不要求掌握" | 高 |
| 区分度预警 | 明确说这是拉开分数差距的地方 | "这是拉开差距的地方" | 高 |
| 出题逻辑 | 解释出题策略、A/B 卷差异、分值分配 | "两张卷子分量要相当""B 卷才会考这个" | 高 |
| 答题策略 | 教你怎么答能拿分、有什么坑 | "不能光写干条条，要展开""审题要看清" | 中 |

第二层同样设置人工判定出口——"考试信号"内部也可能出现无法归入以上七类的模糊样本。

## 三、切片精度：渐进三层

分类体系解决"分什么"的问题，切片精度解决"在哪个粒度上分"的问题。

### 为什么句子级扁平分类必然失败

**意图不是句子的属性，而是交际动作的属性。** 一个交际动作通常由多个句子共同完成。把每个句子单独拿出来判意图，相当于把一幅画的每个像素单独拿出来判"这是在画天空还是画人脸"——很多像素本身是模棱两可的，只有拼在一起才有意义。

以一个实际段落为例：

> 句A：接下来接下来就是我们这门课程的重点了。  
> 句B：我们这门课程的重点就是第二章。  
> 句C：第二章里头谈到了什么？  
> 句D：谈到了这个智能装备的分类和主要的功能，当然咱先要了解它的分类。  
> 句E：请问智能装备怎么分？  
> 句F：是不是？  
> 句G：首先我们会知道它分了加工装备、工艺装备，仓储运输装备和辅助装备。

如果逐句判断，A/B/C/F 会被判为"过渡句"或"语气词"而丢弃，只留下 D 和 G。但 D 和 G 单独看，只是在"复述知识"——考试信号消失了。

原因在于：**信号不在 D 和 G 里，信号在 A+B 里。** 这 7 句话实际上是同一个交际动作：

- **[铺垫] A+B**："重点是第二章" → 范围划定（这是最强的考试信号之一）
- **[设问] C+E+F**："谈到了什么？怎么分？" → 引导注意力
- **[核心] D+G**："分类和主要功能，四大类" → 具体考点

扁平分类只看到了句子的形式，错过了交际动作的意图。正确的做法是先把句子合并为意图单元，再对单元做分类。

### 三层结构

| | 第一层 | 第二层 | 第三层 |
|--|--------|--------|--------|
| 做什么 | 话题在哪切换 | 老师在干什么 | 具体考点是什么 |
| 处理单元 | 全文 | 30-50 句的段落 | 3-10 句的意图单元 |
| 判断类型 | 边界检测 | 意图粗分类 | 结构化精提取 |
| 用什么 | 规则 + embedding | 小模型批处理 | 大模型精提取 |
| 成本 | 几乎为零 | 低 | 高（但只覆盖约 20-40% 内容） |

### 第一层：段落级（话题切分）

把全文按主题切分为 15-20 个大段，不做任何价值判断。切分依据是老师口语中的显性边界信号（"接下来我们讲""好，第二个问题""最后一个"）以及相邻句子语义相似度的骤降点。这一层不需要 LLM，成本几乎为零。

### 第二层：意图段级（交际动作识别 + 粗分类）

在每个段落内，把连续句子合并为"意图单元"，再对单元做粗分类（考试信号 / 知识内容 / 噪声 / 人工判定）。合并的依据是课堂口语的交际节奏模式：

- **模式A**：[设问] → [核心回答] → [补充举例] → 通常对应必考点名
- **模式B**：[提醒注意] → [陈述规则] → [举例] → 通常对应答题策略
- **模式C**：[铺垫] → [下结论/划范围] → 通常对应范围信号
- **模式D**：[闲聊] → [展开] → [收尾] → 通常对应噪声

这一层只做粗分类，不做 7 小类的精细判断，使用小模型批处理，成本低。

### 第三层：句子级（精细提取）

只对前两层筛出的"考试信号"意图单元执行。这一层才真正关心细节——考点对应什么题型、A 卷还是 B 卷、要不要展开叙述、有没有"了解即可"的内容混在里面。提取 schema 如下：

```json
{
  "知识点": "具体考点名称",
  "关联题型": "名词解释 / 填空 / 简答 / 综合分析",
  "确定性": "必考 / 大概率 / 可能 / 排除",
  "卷面": "A卷 / B卷 / 通用",
  "答题要点": "老师提到的具体答题要求",
  "关联知识点": "跨章节融合线索",
  "原文位置": "句子索引区间"
}
```

大模型的高成本推理只用在这一层，但由于此时处理的只是约 20-40% 的原始内容，整体成本依然可控。

## 四、完整执行 SOP

### 步骤 1：硬过滤——删除确定性噪声

用正则和关键词直接删除，零成本去掉约 30% 的内容：

| 类别 | 匹配方式 |
|------|----------|
| 录音元数据 | 日期时间戳、Recorder 前缀 |
| 课堂动作指令 | "翻到第X页""抬头看黑板""等我一下" |
| 纯语气词（≤4字且无实词） | "好。""对吧？""是不是？""嗯。" |
| 独立过渡引导语 | 以"好，我们继续""接下来""来，"开头且 ≤10 字的句子 |
| 明确闲聊段 | 连续 3 句以上谈论个人/社会话题且不含课程术语 |

**注意**：硬过滤只删除"确定性"噪声。"接下来这是我们课程的重点"虽然形式上是过渡句，但意图是范围划定，不应被删除。硬过滤之后需要抽查 10 条，确认没有误删携带考试信号的句子。

### 步骤 2：话题切分（第一层）

扫描显性切换词，无标记区则计算相邻句对的 embedding 余弦相似度（可用 text2vec-large-chinese），骤降处即为边界。每段给出简短主题标签，不做价值判断。

### 步骤 3：意图单元合并 + 粗分类（第二层）

检测到意图模式起始标记时开始合并，遇到收尾信号或新主题时截断，形成意图单元。对每个单元做 4 选 1 粗分类（考试信号 / 知识内容 / 噪声 / 人工判定）。置信度低于 0.8、或 top-2 得分差小于 0.1 的单元，归入人工判定桶。

对判定为"考试信号"的单元，继续做 7 选 1 细分类（题型号令 / 必考点名 / 正范围信号 / 负范围信号 / 区分度预警 / 出题逻辑 / 答题策略）。

### 步骤 4：精细提取（第三层）

只对"考试信号"单元执行，用大模型按上文 schema 逐一提取。

### 步骤 5：人工判定闭环

人工判定结果写入 few-shot 库，下次同类自动分类。目标是 3 次迭代后人工判定比例降至 5% 以下。

### 步骤 6：输出考试地图

按考试价值重组，不按录音顺序排列：

```
🟢 必考层    知识点 | 题型 | 卷面 | 答题要点
🟡 高概率层  ...
🟠 可能层    ...
🔴 排除层    知识点 | 不考原因 | 原文依据
⚡ 陷阱区    知识点 | 常见错误
🔗 融合区    知识组合 | 可能的综合题形式
❓ 待确认    原文 | 分类器猜测 | 阻塞原因
```

## 五、常见陷阱

1. **过渡句 ≠ 噪声**。"接下来这是我们课程的重点"——形式是过渡，意图是范围划定。硬过滤规则要对 ≤10 字的过渡句设长度门槛，带实质内容的不能删。

2. **负信号比正信号更容易漏**。老师说"其他了解一下就行"，语速快、音量低，ASR 可能识别模糊。排除层是最容易出现遗漏的地方，需要专项抽查。

3. **设问句不要单独判断**。"这个地方会考什么？"单独看没有任何信号，必须和后续回答合并为一个意图单元才能正确分类。

4. **同一知识点多次出现 ≠ 强调**。闲聊、课堂管理、划重点三种语境中都可能出现同一个术语，只有划重点语境中出现才是考试信号。

5. **输出完整 ≠ 信号准确**。最终的考试地图要做两层验证：对照课件验证知识内容是否准确，对照常识验证信号判断是否合理。完整漂亮的文档和高命中率的考试地图是两件事。

## 六、与 PPT/课件的关系

录音和 PPT 不是等权信息源：

| 信息源 | 角色 | 处理方式 |
|--------|------|----------|
| 录音 | 决定考试范围 | 提取考试信号，生成考试地图 |
| PPT/课件 | 填充知识内容 | 对录音提取出的考点做内容验证和补充 |

具体规则：

- 录音中说"这个会考" + 课件中是大标题 → 必考层
- 录音中没提 + 课件中是大标题 → 可能层（开卷考试要重点覆盖）
- 录音中说"了解一下就行" + 课件中是正文 → 排除层
- 录音中强调"这是陷阱" + 课件中有对比内容 → 陷阱区

## 七、与开卷/闭卷策略的结合

本方法论生成的是**考试地图**。如何把考试地图变成考场资料，取决于考试形式：

- **开卷考试**：对 🟢🟡🟠 层都要覆盖，重点写完整答案和索引。
- **闭卷考试**：向 🟢🟡 层倾斜，🟠 层了解即可，但仍需评估。

详见 `开卷策略.md` 和 `闭卷策略.md`。
