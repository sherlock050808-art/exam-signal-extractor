# 开卷闭卷考试助手 v3

当用户需要准备考试、整理考点、生成考场资料时，使用本 Skill。

v3.0 核心升级：
1. 输入先分为**考试意图**和**考试内容**两类，分别处理。
2. 用 **Knowledge Node** 承载课程结构，节点直接来自 PPT/教材/大纲标题。
3. 节点是**查询入口 + 意图标签载体**，不是内容容器。
4. 输出模板直接复用 v2.0 已验证的闭卷/开卷双策略。

---

## 一、触发条件

以下任一情况即触发：

- 用户提到"考试复习""考前冲刺""开卷考试""闭卷考试"
- 用户提供课堂录音、课件 PDF、随堂练习、考点文件
- 用户要求整理考点、生成小抄、生成考场资料
- 用户要求做名词解释、简答题、综合分析题整理
- 用户要求建立知识树、Knowledge Node、考试资料自动化生成

---

## 二、触发后第一步：确认考试形式

**必须在整理任何内容之前确认考试形式。** 闭卷与开卷的资料形态、输出结构、记忆策略完全不同。

若用户已明确说明，直接按对应策略执行；若未说明，主动询问：

> "这场考试是开卷还是闭卷？两种形式的复习资料结构差别很大，我先确认一下。"

| 确认结果 | 立即决定的事项 |
|---------|---------------|
| 闭卷 | 采用"题型分布前置 + 按章节展开 + box体系 + 速查索引"结构 |
| 开卷 | 采用"考试地图 + 完整答案 + 强索引"结构 |
| 待定 | 先生成通用知识树和考试地图，待确认后再派生具体资料 |

## 四、两条处理线的职责

### 4.1 考试意图处理线（v2.0）

负责回答：**考什么？重不重要？什么题型？什么不考？**

| 输入 | 处理方式 | 输出 |
|------|---------|------|
| 课堂录音 | 六步工作流：硬过滤→话题切分→意图单元合并+粗分类→精细提取→课件对照验证→输出考试地图 | 结构化信号 |
| 考点大纲 | 提取标题级考点 | 范围确认 |
| 往年真题 | 统计高频考点 | 历史验证 |
| 旧版资料信号 | 提取老师原话、重点标记 | 已沉淀信号 |

**核心原则**：
- 信号 > 内容
- 录音决定范围
- 排除层 = 必考层
- 直接分类，**不量化概率**

### 4.2 考试内容处理线（Knowledge Node）

负责回答：**这个知识点是什么？在资料哪里？和什么相关？**

| 输入 | 处理方式 | 输出 |
|------|---------|------|
| 课件 PDF/PPT | 解析标题、加粗、独立条目 | 课程结构 |
| 教材 | 小节标题、定义、公式、例题 | 知识补充 |
| 旧版复习资料 | 完整答案、例题、陷阱总结 | 可直接使用的内容 |

**核心原则**：
- 节点直接来自 PPT/教材/大纲的已有结构
- 不人工切分节点
- 节点只存定位和标签，正文在生成时从原始资料拉取
- **必须定位原文**：生成任何答案之前，必须先根据 `ppt_source` / `textbook_source` 找到原始资料中的对应内容，禁止直接根据 `summary` 或自己的知识编造答案

## 五、Knowledge Node Schema

```yaml
---
# ── 考试内容定位（从 PPT/教材/大纲直接提取）──
id: cnc_closed_loop
name: 闭环控制
chapter: 1
section: 1.4
summary: 直接检测工作台实际位置并反馈，精度高但系统复杂。
keywords: [闭环, 反馈]

ppt_source: 第1章 -绪论.pdf p.33
textbook_source: 教材第X章第X节
syllabus_source: 考点大纲 三、数控机床的控制方式
related_nodes: [cnc_open_loop, cnc_semi_closed_loop]

# ── 考试意图标签（从录音/大纲/真题映射而来）──
exam_layer: 必考层                    # 必考/高概率/可能/排除/待确认
question_types: [名解, 简答, 分析]
teacher_signal: 开环、闭环、半闭环是非常重要的考点，往年都出现过。
recording:
  - timestamp: "10:15"
    note: 开环、闭环、半闭环是非常重要的考点
past_exam: true

# ── 特殊标记 ──
traps:
  - trap: 容易与半闭环混淆
    clarification: 半闭环检测电机轴或丝杠，闭环检测工作台实际位置
fusions:
  - with: [cnc_open_loop, cnc_semi_closed_loop]
    question_form: 比较三种控制方式
---
```

**重要**：节点不存储完整正文内容。生成资料时，必须根据 `ppt_source` / `textbook_source` 去原始资料中定位原文，再编写答案。**禁止直接根据 `summary` 编造答案，禁止把未经验证的内容写进闭卷资料。**

## 六、融合层规则

| 考试意图信号 | 对节点的操作 |
|-------------|------------|
| 老师说"这个必考" | `exam_layer = 必考层` |
| 老师提到但没强调 | `exam_layer = 高概率层` |
| 大纲有但录音没提 | `exam_layer = 可能层` |
| 老师说"不用看" | `exam_layer = 排除层` |
| 往年反复考 | `past_exam = true` |
| 老师说"容易错" | 添加 `traps` 标记 |
| 老师说"把 A 和 B 结合看" | 添加 `fusions` 标记 |
| 题型信号 | 写入 `question_types`；无明确信号时按内容特征默认推断，标记 `[内容推断]` |
| 信息不足 | `exam_layer = 待确认` |

## 七、输出层（复用 v2.0 模板）

### 7.1 闭卷考试

采用"题型分布前置 + 按章节展开 + box体系 + 速查索引"结构：

```
1. 封面 + 使用说明 + 闭卷核心策略
2. 题型分布（按 question_types 自动列出节点）
3. 考试情报（teacher_signal 汇总）
4. 按章节展开的知识内容
   ├── 知识点标题（name + PPT/教材/大纲页码）
   ├── 课堂信号框（teacher_signal / recording）
   ├── 重点框（从原始资料定位原文，按 exam_layer 裁剪）
   ├── 易错框（traps）
   ├── 融合区框（fusions）
   └── 公式框/例题框（需要时从原始资料提取）
5. 按题型速查索引
6. 附录：公式表 / G-M代码表
```

### 7.2 开卷考试

采用"考试地图 + 完整答案 + 强索引"结构：

```
1. 考试地图（按 exam_layer 七层结构）
2. 名词解释清单
3. 简答题（采分点 + 完整答案）
4. 综合分析题（完整可抄答案）
5. 强索引（章节/题型/关键词）
```

## 八、执行步骤

当用户说"按这个帮我准备某某考试"时，按以下步骤执行。

**重要前置条件**：在处理录音之前，必须先读取 `references/从课堂录音中提取考试信号-完整方法论.md`，严格按 v2.0 六步工作流执行。禁止跳过任何一步。

### 步骤 1：确认考试形式

询问开卷/闭卷。若用户已说明，跳过。

### 步骤 2：收集资料

要求用户提供：
- 课件 PDF/PPT（用于建知识树）
- 课堂录音转写（用于提取意图）
- 考点大纲（用于校准）
- 往年真题/作业（可选，用于验证）
- 旧版复习资料（可选，用于补充完整答案和例题）

### 步骤 3：建立 Knowledge Node 知识树

1. 解析 PPT/教材/大纲的标题结构
2. 每个标题直接生成一个 Knowledge Node
3. 填写考试内容定位字段：id / name / chapter / section / summary / keywords / ppt_source / related_nodes

### 步骤 4：处理考试意图（必须严格按 v2.0 六步工作流）

**必须先读取 `references/从课堂录音中提取考试信号-完整方法论.md`，再执行以下 6 步：**

1. **硬过滤**：删除录音中的确定性噪声（元数据、课堂动作、纯语气词、无关闲聊等）。
2. **话题切分**：按切换词或 embedding 相似度骤变处，把录音切成 15-20 个主题段落。
3. **意图单元合并 + 粗分类**：把连续句子合并为完成一个交际动作的"意图单元"，粗分为考试信号、知识内容、噪声、人工判定四类。
4. **精细提取**：对考试信号单元提取结构化信息：知识点名称、关联题型、确定性、答题要点、关联知识点、原文定位。
5. **课件对照验证**：把课件标题级内容与录音信号对照，课件有但录音未提的升级为"可能层"。
6. **输出考试地图**：按七层结构（必考/高概率/可能/排除/陷阱/融合/待确认）重组信号。
7. **题型重建**：从所有考试信号中全局重建题型清单。无明确信号时按内容特征默认推断，标记 `[内容推断]`。

完成上述 7 步后，再进入步骤 5。

### 步骤 5：融合映射

把考试意图信号映射到对应 Knowledge Node：
- 更新 `exam_layer`
- 更新 `question_types`（来自题型重建步骤；无明确信号时按内容特征默认推断，标记 `[内容推断]`）
- 添加 `teacher_signal` / `recording`
- 添加 `past_exam` / `traps` / `fusions`

### 步骤 6：生成资料（强制遵循 MD 模板 + 定位原文）

**⚠️ 生成前必须先读取模板文件：**

- 闭卷：**必须先读取 `templates/闭卷资料MD模板-v3.md`**，再按模板的 6 个固定 Section 组装内容。
- 开卷：当前版本优先使用闭卷模板生成基础内容，开卷专用 MD 模板待补充。生成后按开卷需求增加完整答案和强索引即可。

**MD 模板的约束力：**
- 模板定义了最终 MD 的**顶层 Section 数量和顺序**——不允许增删改。
- 模板定义了**Box 类型**（仅 4 种：课堂信号框/重点框/易错框/公式框）——不允许发明新 Box。
- 模板定义了**内容详略规则**（必考=完整/高概率=采分点/可能=3~5条无box/排除=不输出）——不允许违反。
- 模板定义了**节点渲染管道**（标题→课堂信号→重点→易错→融合→公式/代码）——不允许乱序。

**输出 MD 后必须自检**：对照模板 §5.0 渲染规则，逐项检查以下硬性约束：

1. 顶层 Section 是否只有 6 个（封面、使用说明、题型分布、考试情报、章节内容、速查索引）？
2. 是否不存在"考试地图""真题验证"等非法章节？
3. 是否只使用了 4 种 Box？
4. 🟠可能层是否只有 3-5 条要点（没有使用 box）？
5. ⚪排除层是否没有输出正文？
6. 每个节点标题是否包含 PPT 页码来源？

根据考试形式选择模板：
- 闭卷：生成题型分布 + 按章节展开 + 速查索引
- 开卷：生成考试地图 + 完整答案 + 强索引

**生成每个知识点的答案时，必须执行以下子步骤：**

1. **读取节点**：获取 `ppt_source` / `textbook_source` / `syllabus_source`。
2. **定位原文**：打开对应原始资料（PPT PDF、教材、旧版资料），找到该知识点所在位置。
3. **提取内容**：从原始资料中提取定义、分类、公式、例题、老师原话等。
4. **结合意图裁剪**：根据 `exam_layer` 决定详略：
   - 必考层：完整答案，可默写
   - 高概率层：采分点 + 简要展开
   - 可能层：一句话识别
   - 排除层：不输出正文
5. **标注来源**：在答案后标注来源，如 【PPT p.33】【教材第 X 章】。

**禁止行为**：
- ❌ 禁止跳过 MD 模板直接生成资料（必须先读取 `templates/闭卷资料MD模板-v3.md`）
- ❌ 禁止在 MD 中新增模板未定义的顶层 Section（如"考试地图""真题验证"）
- ❌ 禁止发明模板规定之外的 Box 类型
- ❌ 禁止违反内容详略规则（可能层写成长篇、排除层输出正文等）
- ❌ 禁止直接根据 `summary` 编写答案
- ❌ 禁止把 agent 自己的知识直接写进资料
- ❌ 禁止把正文直接塞进闭卷资料而不标注来源
- ❌ 禁止使用"根据一般理解""通常来说"等未经验证的表述

## 十、常见陷阱

1. **不要过度约束 agent，但要守住核心检查点**：给出目标、边界和验收标准，让 agent 自己选择合适的实现方式。例如发现表格超宽时，只需告诉它"修复表格超宽问题"，不必规定必须用 `tabularx` 还是横向页面。约束太死会降低 agent 处理复杂排版问题的能力。

2. **不要跳过 v2.0 六步工作流**：v3.0 的"考试意图处理线"不是一句口号，必须完整执行硬过滤→话题切分→意图单元合并+粗分类→精细提取→课件对照验证→输出考试地图。只粗略扫一遍录音会漏掉关键信号。
3. **不要根据 summary 编造答案**：`summary` 只是用于快速识别的一句话，不能作为答案来源。必须根据 `ppt_source` / `textbook_source` 定位到原始资料原文后再编写答案。
4. **不要标注无法定位的来源**：每个答案、公式、例题后面的来源标注必须能唯一确定地定位到原始资料。`【PPT p.40】` 在存在多个课件时是有歧义的；应写 `【第一章 PPT p.40】` 或 `【项目7 PPT p.40】`。来源标注的核心目标是：不依赖上下文就能打开正确的文件、找到正确的页码/章节。
5. **不要量化概率**：v3.0 不用 `P=0.95` 这种东西，直接分类为必考/高概率/可能/排除。
6. **节点不要存完整内容**：节点只存定位和标签，正文从原始资料拉取。
7. **不要人工划分节点**：PPT/教材/大纲怎么分，节点就怎么分。
8. **不要默认考试形式**：开卷和闭卷输出结构完全不同，必须先确认。
9. **不要漏掉负信号**："了解一下就行""不要求掌握"要标为排除层。
10. **不要把课件内容当意图**：课件大标题是内容来源，老师的话才是意图来源。
11. **输出模板不要重新发明**：直接复用 v2.0 闭卷/开卷双策略。
12. **不要跳过 MD 模板**：生成闭卷资料 MD 前必须读取 `templates/闭卷资料MD模板-v3.md`，生成后必须逐项自检。跳过模板自由发挥是过去输出质量差的首要原因。
13. **不要把中间产物写进最终资料**：考试地图、真题验证表、课件截图索引是工作过程的中间产物，不得作为最终 MD 的独立 Section。
