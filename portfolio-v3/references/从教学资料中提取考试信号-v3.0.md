> 版本：v3.0
> 核心思想：考试意图和考试内容分开处理；Knowledge Node 是查询入口，不是内容容器；输出模板直接复用 v2.0。

# 从教学资料中提取考试信号（v3.0）

## 一、问题重新定义

准备考试需要回答两个问题：

1. **考试意图**：考什么？重不重要？什么题型？什么不考？
2. **考试内容**：这个知识点是什么？在资料哪里？和什么相关？

v3.0 把这两个问题分开处理，用两条并行工作流解决，最后在节点上融合。

---

## 三、输入层：先分类

### 考试意图来源

负责回答"考什么、重不重要"。

| 来源 | 处理方式 | 输出 |
|------|---------|------|
| 课堂录音 | v2.0 六步工作流：硬过滤→切分→意图单元→精细提取 | 结构化信号 |
| 考点大纲 | 提取标题级考点 | 范围确认 |
| 往年真题 | 统计高频考点 | 历史验证 |
| 旧版资料中的信号 | 提取老师原话、重点标记 | 已沉淀信号 |

### 考试内容来源

负责回答"是什么、在哪、关联谁"。

| 来源 | 处理方式 | 输出 |
|------|---------|------|
| 课件 PDF/PPT | 解析标题、加粗、独立条目 | 课程结构 |
| 教材 | 小节标题、定义、公式、例题 | 知识补充 |
| 旧版复习资料 | 完整答案、例题、陷阱总结 | 可直接使用的内容 |

## 四、考试意图处理线（v2.0 方法）

### 步骤 A1：硬过滤

删除确定性噪声：
- 录音元数据
- 课堂动作指令（"翻到第 X 页"）
- 纯语气词（"好""对吧"）
- 独立过渡引导语（≤10 字）
- 与课程无关的闲聊

注意：承载考试信号的过渡句不能删，如"接下来这是我们课程的重点"。

### 步骤 A2：话题切分

按显性切换词或 embedding 相似度骤降处切分，产出 15-20 个主题段落。

常见切换词：
- "接下来我们讲……"
- "第二个问题……"
- "最后一个……"
- "另外还有……"

### 步骤 A3：意图单元合并 + 粗分类

把连续句子合并为完成一个交际动作的"意图单元"，粗分为四类：

| 大类 | 定义 | 下一步 |
|------|------|--------|
| 考试信号 | 传递考试相关信息 | → 步骤 A4 |
| 知识内容 | 纯知识讲解，未涉考试 | → 丢弃或留作参考 |
| 噪声 | 闲聊/管理/无关 | → 丢弃 |
| 人工判定 | 置信度低 | → 人判后入 few-shot 库 |

常见交际模式：

| 模式 | 结构 | 典型意图 |
|------|------|----------|
| A | [设问] → [核心回答] → [补充举例] | 必考点名 |
| B | [提醒注意] → [陈述规则] → [举例] | 答题策略 |
| C | [铺垫] → [下结论/划范围] | 范围信号 |
| D | [闲聊] → [展开] → [收尾] | 噪声 |

### 步骤 A4：精细提取

对考试信号单元提取结构化信息：

```json
{
  "知识点": "具体考点名称",
  "关联题型": "名词解释 | 填空 | 简答 | 综合分析",
  "确定性": "必考 | 高概率 | 可能 | 排除",
  "卷面": "A卷 | B卷 | 通用",
  "答题要点": "老师提到的具体答题要求",
  "关联知识点": ["跨章节关联的其他考点"],
  "原文定位": "句子索引区间"
}
```

## 五、考试内容处理线（Knowledge Node）

### 节点来源

节点不是人工划分出来的，而是从已有结构化内容中直接提取：

| 来源 | 节点是什么 |
|------|-----------|
| PPT 标题 | 每一页/每一节的标题 |
| 教材小节标题 | 1.1、1.1.1 这样的小节名 |
| 考点大纲条目 | 大纲中的"三、数控机床的控制方式" |
| 旧版资料中的章节标题 | 已经沉淀的知识结构 |

### 节点粒度原则

> PPT/教材/大纲怎么分，节点就怎么分。

不需要纠结"这个概念够不够独立成节点"。如果某个标题下内容太多，拆分的依据也是 PPT/教材本身的子标题，不是我们自己创造。

### 简化 Node Schema

```yaml
---
# ── 考试内容定位（从 PPT/教材/大纲直接提取）──
id: cnc_closed_loop
name: 闭环控制
chapter: 1
section: 1.4
summary: 直接检测工作台实际位置并反馈，精度高但系统复杂。
keywords: [闭环, 反馈]

ppt_source: 第1章 -绪论.pdf p.33
textbook_source: 教材第X章第X节
syllabus_source: 考点大纲 三、数控机床的控制方式
related_nodes: [cnc_open_loop, cnc_semi_closed_loop]

# ── 考试意图标签（从录音/大纲/真题映射而来）──
exam_layer: 必考层
question_types: [名解, 简答, 分析]
teacher_signal: 开环、闭环、半闭环是非常重要的考点，往年都出现过。
recording:
  - timestamp: "10:15"
    note: 开环、闭环、半闭环是非常重要的考点
past_exam: true

# ── 特殊标记 ──
traps:
  - 容易与半闭环混淆：半闭环检测电机轴或丝杠，闭环检测工作台
fusions:
  - with: [cnc_open_loop, cnc_semi_closed_loop]
    question_form: 比较三种控制方式
---
```

### 节点不是内容容器

| 旧理解（v3.0 之前） | v3.0 理解 |
|------------------|----------|
| 节点的 `body` 存放完整内容 | 节点只存放**定位信息**和**意图标签** |
| 生成时直接读 `node.body` | 生成时必须根据 `ppt_source` / `textbook_source` 去原始资料里找原文 |
| 根据 `summary` 一句话就能写答案 | `summary` 只能用于识别，**禁止**作为答案来源 |
| 节点越来越臃肿 | 节点保持轻量，内容在原始资料里 |

**核心禁令**：生成任何答案、公式、例题之前，必须先定位到原始资料中的对应位置。未标注来源的内容不能进入最终资料。

## 六、融合层

### 映射规则

| 考试意图信号 | 对 Knowledge Node 的操作 |
|-------------|------------------------|
| 老师说"这个必考" | `exam_layer = 必考层` |
| 老师提到但没强调 | `exam_layer = 高概率层` |
| 大纲有但录音没提 | `exam_layer = 可能层` |
| 老师说"不用看" | `exam_layer = 排除层` |
| 往年反复考 | `past_exam = true` |
| 老师说"容易错" | 添加 `traps` 标记 |
| 老师说"把 A 和 B 结合看" | 添加 `fusions` 标记 |
| 信息不足 | `exam_layer = 待确认` |

### 冲突处理

| 冲突场景 | 处理原则 |
|---------|---------|
| 老师说"不考"但大纲有 | 优先老师信号，标为排除层，同时记录冲突 |
| 老师说"必考"但往年没考 | 标为必考层，老师信号优先 |
| 大纲有但录音明确排除 | 标为排除层，记录依据 |
| 多个信号强度不同 | 以最强信号为准，其他作为参考 |

## 七、输出层（直接复用 v2.0 模板）

### 闭卷资料

模板来源：v2.0 `闭卷策略.md` + 旧版 `数控技术复习指导-闭卷冲刺版.tex`

```
1. 封面 + 使用说明 + 闭卷核心策略
2. 题型分布（按 question_types 列出节点）
3. 考试情报（teacher_signal 汇总）
4. 按章节展开的知识内容
   │
   └── 对每个 Knowledge Node，执行：
       │
       a. 知识点标题
       │   └── name + 【PPT p.xx】+ 【教材 p.xx】+ 【大纲 xxx】
       │
       b. 课堂信号框
       │   └── 来源：teacher_signal / recording
       │       老师原话 + 录音时间戳
       │
       c. 重点框 / 名解答 / 简答答案 / 分析答案
       │   └── 来源：根据 ppt_source / textbook_source
       │       去原始资料中定位原文
       │       再按 exam_layer 和 question_types 编写/裁剪
       │       必考层 → 完整答案
       │       高概率层 → 采分点 + 简要答案
       │       可能层 → 一句话识别
       │       排除层 → 不输出正文
       │
       d. 易错框
       │   └── 来源：traps，可结合原文中的对比/反例
       │
       e. 融合区框
       │   └── 来源：fusions + related_nodes
       │       把相关节点原文组合成综合题答案
       │
       f. 公式框 / 例题框
           └── 来源：根据 ppt_source / textbook_source
               去原始资料中定位公式和例题
               按 question_types 中的"计算"/"编程"决定是否输出
5. 按题型速查索引
6. 附录：G/M代码速查表、公式表
```

### 开卷资料

模板来源：v2.0 `开卷策略.md` + Portfolio 模板

```
1. 考试地图（按 exam_layer 七层结构）
2. 名词解释清单
   └── 来源：根据节点 name + summary + 原文编写
3. 简答题
   └── 来源：根据节点 question_types=简答，定位原文编写完整答案
4. 综合分析题
   └── 来源：根据 fusions + 相关节点原文编写完整可抄答案
5. 强索引（章节/题型/关键词）
```

## 九、核心原则

1. **输入先分类**：考试意图和考试内容分开处理。
2. **意图线用旧版**：录音/大纲/真题决定考什么、重不重要，直接分类，不量化概率。
3. **内容线用新版**：从 PPT/教材/大纲的已有结构中提取 Knowledge Node，不人工切分。
4. **节点是查询入口**：节点存放定位和意图标签，正文内容在生成时根据来源从原始资料拉取。
5. **输出用旧版模板**：闭卷/开卷双策略直接复用 v2.0 已验证结构。
6. **排除层 = 必考层**：知道什么不考和知道什么会考同样重要。
7. **意图在交际动作层面**：单个句子无法独立表达意图，必须先合并为意图单元再分类。
8. **复盘是改进来源**：每次考试后必须复盘，修正节点分类和内容定位。

---

## 十、与 v2.0 的关系

v3.0 不是推翻 v2.0，而是对 v2.0 的扩展：

- **保留**：v2.0 的六步信号提取、七层考试地图、闭卷/开卷双策略输出模板。
- **新增**：Knowledge Node 作为内容查询入口、输入分类、节点化内容管理。
- **修正**：不再把概率作为核心组织方式；节点不存储完整内容而是定位到原始资料。

v2.0 解决"怎么判断考试重点"，v3.0 在 v2.0 基础上解决"怎么持续维护和生成资料"。
