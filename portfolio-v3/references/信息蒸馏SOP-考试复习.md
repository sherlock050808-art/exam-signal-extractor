# 信息蒸馏 SOP：从课堂录音到考试地图

> **核心原则**：目标不是总结知识，是蒸馏考试信号。知识密度 ≠ 考试价值。老师的态度和暗示（占文本 ~10%）对考题的预测力远大于知识内容本身（占文本 ~90%）。

---

## 一、目标产出

不是复习笔记，是**考试地图**。包含六个层次：

| 层次 | 内容 | 说明 |
|------|------|------|
| 🟢 必考层 | 知识点 + 题型 + 卷面 + 答题要点 | 老师明确指认的考点 |
| 🟡 高概率层 | 同上，但确定性稍低 | 老师暗示/强调但未直接说"必考" |
| 🟠 可能层 | 知识点 + 可能题型 | 正范围信号覆盖但无直接题型指示 |
| 🔴 排除层 | 知识点 + 不考原因 + 原文依据 | 负范围信号覆盖——和必考层同等重要 |
| ⚡ 陷阱区 | 知识点 + 常见错误 + 区分方式 | 区分度预警信号覆盖 |
| 🔗 融合区 | 知识点组合 + 可能的综合题形式 | 跨章节关联信号覆盖 |
| ❓ 待确认 | 原文 + 分类器猜测 + 阻塞原因 | 🔶 需课件对照 / 🔷 需人工判定 |

---

## 二、分类体系：两层递进

### 第一层：粗分类（4 类）

不做精细判断，只回答一个问题：**这段话和考试有没有关系？**

| 大类 | 定义 | 分流方向 |
|------|------|----------|
| 考试信号 | 老师在传递关于考试的任何信息 | 进入第二层细分类 |
| 知识内容 | 老师在讲知识本身，未涉及考试 | 进入课件对照验证通道 |
| 噪声 | 与教学无关的一切 | 丢弃 |
| ⚡ 人工判定 | 无法自动归入以上三类的模糊边界情况 | 标记，交由人工判断 |

**人工判定桶的设计原则**：

- 分类器对任何置信度低于阈值（建议 0.8）的样本，不强判，不丢弃，统一归入人工判定桶
- 人工判完之后，该样本自动成为 few-shot 例句，提升后续分类准确率
- 这个桶同时是**质量兜底**和**持续改进机制**——目标：3 次迭代后人工判定比例 < 5%

### 第二层：细分类（仅对"考试信号"展开，7 类）

回答一个问题：**老师在传递哪一类考试信号？**

| 意图类别 | 定义 | 典型触发模式 | 考试价值 |
|----------|------|-------------|:---:|
| 题型号令 | 明确说明题型、题量、分值、考试结构 | "有名词解释、填空、简答、综合分析" | 最高 |
| 必考点名 | 明确指认某知识点会考、以什么形式考 | "这个一定会考名词解释""我肯定考" | 最高 |
| 正范围信号 | 暗示/明示需要重点关注 | "注意一下""这个地方小心""要掌握" | 高 |
| 负范围信号 | 明确/暗示不考或只需了解 | "了解一下就行""不要求掌握""其他看看就好" | 高 |
| 区分度预警 | 明确说这是拉开分数差距的地方 | "这是拉开差距的地方""分数差距就在这里" | 高 |
| 出题逻辑 | 解释出题策略、A/B 卷差异、分值分配原因 | "两张卷子分量要相当""B 卷才会考这个" | 高 |
| 答题策略 | 教你怎么答能拿分、有什么坑 | "不能光写干条条，要展开""审题要看清" | 中 |

第二层分类同样设置人工判定出口。考试信号内部也可能出现无法归入以上七类的模糊样本。

## 三、切片精度：渐进三层

**核心原则**：意图不在句子层面，而在交际动作层面。一个交际动作通常由多句共同完成。必须先把句子**合并为意图单元**，再对单元做分类。

### 第一层：段落级（话题切分）

**做什么**：把全文按主题切分为 15-20 个大段。

**怎么切**（不依赖 LLM）：

1. 扫显性切换词——"接下来我们讲""第二个问题""最后一个""另外还有""好了"
2. 无标记区——计算相邻句对 embedding 余弦相似度（text2vec-large-chinese），骤降处 = 边界
3. 每段给简短主题标签，不做价值判断

**成本**：几乎为零（规则 + embedding 相似度）。

### 第二层：意图段级（交际动作识别 + 粗分类）

**做什么**：在段落内，把连续句子合并为"意图单元"，判这个单元的意图大类。

**合并依据**：课堂口语有固定的交际节奏模式。一个意图单元的完整结构通常是：

```
模式 A: [设问] → [核心回答] → [补充举例]         → 必考点名
模式 B: [提醒注意] → [陈述规则] → [举例]           → 答题策略
模式 C: [铺垫] → [下结论"这是重点"]                → 范围信号
模式 D: [闲聊引入] → [展开] → [收尾"好了我们继续"] → 噪声
```

**关键**：合并之后再分类，而不是逐句分类后再拼。因为单个句子往往无法独立表达意图——"是不是？""为什么这么讲？"单看没有意义，只有放在它的交际动作里才有意义。

**分类粒度**：这一层只做粗分类（考试信号 / 知识内容 / 噪声 / 人工判定），不做 7 小类的精细分类。

**输出**：每个意图单元一个大类标签 + 核心主张的一句话摘要。

**成本**：低（小模型批处理，一次处理一个段落的全部意图单元）。

### 第三层：句子级（精细提取）

**做什么**：只对前两层筛出来的"考试信号"意图单元做结构化提取。

**为什么只有这一层才到句子级**：考试信号的细节（考点对应什么题型、A 卷还是 B 卷、要不要展开叙述）需要精细阅读。但"考试信号"单元只占全文的约 20-40%——大部分内容在第二层已经被分流了。

**提取 schema**：

```json
{
  "知识点": "具体考点名称",
  "关联题型": "名词解释 | 填空 | 简答 | 综合分析",
  "确定性": "必考 | 大概率 | 可能 | 排除",
  "卷面": "A卷 | B卷 | 通用",
  "答题要点": "老师提到的具体答题要求",
  "关联知识点": ["跨章节关联的其他考点"],
  "原文定位": "句子索引区间"
}
```

**成本**：高（大模型精提取），但只覆盖 20-40% 的原始内容。

### 三层对比

| 维度 | 第一层 | 第二层 | 第三层 |
|------|--------|--------|--------|
| 处理单元 | 全文 → 15-20 段 | 每段 → 若干意图单元 | 每个高价值意图单元 |
| 判断类型 | 话题边界检测 | 意图粗分类（4 选 1） | 结构化提取 |
| 用什么工具 | 规则 + embedding | 小模型 + few-shot 批处理 | 大模型精提取 |
| 成本 | ~0 | 低 | 高（但覆盖面小） |

## 四、模型策略：缩小大小模型差距

**原则**：贵的推理只用在值得的地方。

### 成本递进管道

```
全文 100%
  → 硬过滤（正则规则）去掉确定性噪声 → 剩余 ~70%
  → 第一层：规则切分段落 → 零成本
  → 第二层：小模型粗分类 → 低成本，全覆盖
      → 噪声 ~30%：丢弃
      → 知识内容 ~30%：进入课件对照通道
      → 考试信号 ~35%：进入第三层
      → 人工判定 ~5%：人工确认后补入 few-shot 例句库
  → 第三层：大模型精提取 → 高成本，但只覆盖 ~35%
```

### 缩小大小模型差距的五项策略

1. **硬过滤优先**：正则规则消除确定性噪声（30%+），在任何 LLM 调用之前执行
2. **意图单元 ≤ 1000 tokens**：避免长上下文稀释分类准确率
3. **粗分类用小模型**：4 选 1 粗分类对模型能力要求低，7 选 1 细分类仍可用小模型
4. **few-shot > 模型大小**：10 个精选例句对分类准确率的提升超过换一个更大的模型
5. **批处理 30-50 句/次**：避免单句调用的开销，也避免整段一次丢入导致注意力稀释

## 五、完整工作流（6 步）

```
原始录音转录
    │
    ├─→ [步骤1 硬过滤] 正则删除确定性噪声（课堂动作、元数据、纯语气词）
    │    削减 ~30% token
    │
    ├─→ [步骤2 话题切分] 第一层：规则 + embedding 检测话题边界
    │    产出 15-20 个主题段落
    │
    ├─→ [步骤3 意图单元合并+粗分类] 第二层：小模型批处理
    │    ├── 噪声 → 丢弃
    │    ├── 知识内容 → 课件对照验证通道
    │    ├── 考试信号 → 进入步骤 4
    │    └── 人工判定 → 人工确认 → few-shot 库
    │
    ├─→ [步骤4 精细提取] 第三层：大模型对"考试信号"单元做结构化提取
    │    产出每一条考点的：知识点、题型、确定性、卷面、答题要点、原文定位
    │
    ├─→ [步骤5 人工判定闭环] 置信度<0.8 → 标记原文+top-2猜测 → 人判
    │    判定结果 → few-shot 库，下次同类自动分类
    │
    └─→ [步骤6 输出考试地图] 按价值重组，不按录音顺序
         ├── 🟢 必考层
         ├── 🟡 高概率层
         ├── 🟠 可能层
         ├── 🔴 排除层（负信号覆盖的区域）
         ├── ⚡ 陷阱区
         ├── 🔗 融合区（跨章节综合题）
         └── ❓ 待确认（🔶 需课件对照 / 🔷 需人工判定）
```

## 六、各步骤详细说明

### 步骤 1：硬过滤——删除确定性噪声

**目标**：零成本去掉 ~30%，省 token。

**正则 + 关键词直接删**：

| 类别 | 匹配方式 |
|------|----------|
| 录音元数据 | `Recorder -.*`；日期时间戳格式 |
| 课堂动作指令 | "翻到第.*页""抬头看黑板""等我一下""瞄一眼""敲小黑板" |
| 纯语气词（≤4 字且无实词） | "好。""对吧？""是不是？""嗯。" |
| 独立过渡引导语 | 以"好，我们继续""接下来""好了""来，"开头且 ≤10 字的句子 |
| 明确闲聊段 | 连续 3 句以上谈论个人/社会话题且不含课程术语 |

**注意**：承载考试信号的过渡句不能误删——"接下来这是我们课程的重点"形式上像过渡语，但意图是范围划定，需要保留。判断标准：句子是否在划定知识边界或传递考试信息。

### 步骤 2：话题切分（第一层）

**目标**：全文 → 15-20 个主题段落。不做价值判断。

**方法**：

1. 扫显性切换词——"接下来我们讲""第二个问题""最后一个""另外还有""还有一件事"
2. 无标记区——计算相邻句对 embedding 余弦相似度（text2vec-large-chinese），骤降处 = 边界
3. 每段给简短主题标签

**输出**：`[段落 N] 主题标签 | 句子 X-Y`

### 步骤 3：意图单元合并 + 粗分类（第二层）

#### 3.1 合并

一个意图单元 = 老师完成一个完整交际动作。检测到模式起始标记时开始合并，遇到收尾/新主题时截断。

课堂口语常见交际模式：

| 模式 | 结构 | 典型意图 |
|------|------|----------|
| A | [设问] → [核心回答] → [补充举例] | 必考点名 |
| B | [提醒注意] → [陈述规则] → [举例] | 答题策略 |
| C | [铺垫] → [下结论/划范围] | 范围信号 |
| D | [闲聊] → [展开] → [收尾] | 噪声 |

#### 3.2 粗分类（4 选 1）

| 大类 | 定义 | 下一步 |
|------|------|--------|
| 考试信号 | 传递任何考试相关信息 | → 步骤 4 |
| 知识内容 | 纯知识讲解，未涉考试 | → 课件对照验证通道 |
| 噪声 | 闲聊/管理/无关 | → 丢弃 |
| ⚡ 人工判定 | 置信度 < 0.8 或 top2 得分差 < 0.1 | → 人判后入 few-shot 库 |

#### 3.3 细分类（7 选 1）——仅对"考试信号"单元

| 子类 | 典型表述 |
|------|----------|
| 题型号令 | "有名词解释、填空、简答" |
| 必考点名 | "这个一定会考""我肯定出" |
| 正范围信号 | "注意一下""小心""要掌握" |
| 负范围信号 | "了解一下就行""不要求""看看就好" |
| 区分度预警 | "拉开分数差距的地方" |
| 出题逻辑 | "A 卷考这个 B 卷考那个""两张卷子分量要相当" |
| 答题策略 | "不能只写干条条，要展开叙述" |

### 步骤 4：精细提取（第三层）

只对"考试信号"单元执行。用大模型。

**提取 schema**：

| 字段 | 说明 | 可选值 |
|------|------|--------|
| 知识点 | 具体考点名称 | — |
| 关联题型 | 考试形式 | 名词解释 / 填空 / 简答 / 综合分析 |
| 确定性 | 考的可能性 | 必考 / 高概率 / 可能 / 排除 |
| 卷面 | A/B 卷归属 | A卷 / B卷 / 通用 |
| 答题要点 | 老师提到的具体答题要求 | — |
| 关联知识点 | 跨章节关联的其他考点 | — |
| 原文位置 | 句子索引区间 | — |

### 步骤 5：人工判定闭环

- 置信度 < 0.8 或 top2 得分差 < 0.1 → 标记原文 + top-2 猜测 → 提交人工判定
- 判定结果 → 自动补入 few-shot 例句库，下次同类模式自动分类
- **目标**：3 次迭代后人工判定比例 < 5%

**无法判定的分类**：按阻断原因分为两类，分渠道解决。

| 标记 | 阻断原因 | 解决方式 | 判定问题 |
|------|----------|----------|----------|
| 🔶 需课件对照 | 信息不在录音里（老师在引用课件上的视觉内容） | 去课件找到被引用的对象，按课件中的层级判定考点价值 | "给了课件我能判断吗？" → 能 |
| 🔷 需人工判定 | 老师本身没说清（语焉不详、欲言又止、语气模棱两可） | 提交给上过课的人判定，判定理由纳入 few-shot 库 | "给了课件我能判断吗？" → 不能 |

**课件对照的判定规则**：
- 课件中为**标题/加粗/独立条目** → 考点或正范围
- 课件中为**正文/注释/脚注** → 知识复述
- 课件中为**案例/配图/拓展阅读** → 无效或低价值

### 步骤 6：输出考试地图

**按信息价值重组，不按录音顺序排列**。

| 层次 | 包含字段 |
|------|----------|
| 🟢 必考层 | 知识点 + 题型 + 卷面 + 答题要点 |
| 🟡 高概率层 | 同上 |
| 🟠 可能层 | 知识点 + 可能题型 |
| 🔴 排除层 | 知识点 + 不考原因 + 原文依据 |
| ⚡ 陷阱区 | 知识点 + 常见错误 |
| 🔗 融合区 | 知识点组合 + 可能的综合题形式 |
| ❓ 待确认 | 原文 + 分类器猜测 + 阻塞原因（🔶/🔷） |

## 七、信号优先级

### 优先保留（按价值排序）

1. 教师强调内容（"必考""重点""一定要记住"）
2. 易错点与错误预警（"每年都有人在这里栽跟头"）
3. 答题策略（"不能光写干条条，要展开"）
4. 题型说明（"这里会考名词解释"）
5. 范围提示——**正信号和负信号同等重要**（"了解一下就行" = 不考）
6. 知识间关联（"这个和第三章结合考" = 综合题）

### 优先删除

1. 课堂管理信息（"翻到第 X 页""大家能听到吗"）
2. 翻页指令与操作引导
3. 重复性口头语与填充词
4. 与学习目标无关的闲聊
5. 录音元数据

---

## 八、关键区别

- **知识密度 ≠ 考试价值**（两者正交）
- **完整 ≠ 有用**（划掉 80% 不考内容的排除地图 > 覆盖 100% 的完整笔记）
- **排除比覆盖更重要**：知道什么不考比知道什么会考更能节约复习时间
- **意图分类 > 内容分类**：不问"讲了什么知识"，问"老师说这句话在干什么"
- **信号强度有梯度**：第一人称直接确认 > 一般提醒；唯一性信号 > 包含性信号

## 九、质量检查清单

- [ ] 硬过滤是否误删了携带考试信号的句子？（抽查 10 条）
- [ ] 话题切分是否在合理位置？（检查边界处是否有明显应切未切或过度切分）
- [ ] 粗分类准确率 > 90%？（抽查 20 个意图单元）
- [ ] 负范围信号是否全部被捕捉？（排除层最容易遗漏——"了解一下就行"常被语速快、音量低淹没）
- [ ] 人工判定比例是否在下降？（每次迭代后统计 🔶 + 🔷 占比）
- [ ] 考试地图是否包含了所有七个层次？（特别检查排除层和融合区是否为空）
- [ ] 课件对照验证通道的知识内容是否有遗漏的高价值考点？（课件大标题但录音只提了一次的容易漏）

## 十、常见陷阱

1. **过渡句 ≠ 噪声**："接下来这是我们课程的重点"——形式是过渡，意图是范围划定，不能删
2. **负信号比正信号更容易漏**：老师说"其他了解一下就行"语速快、音量低，ASR 可能模糊，需要主动检索负信号特征词（"了解""看看""不要求""不用管"）
3. **设问句不要单独判**："这个地方会考什么？"——和后一句合并才是完整的必考点名动作
4. **同知识点多次出现 ≠ 强调**：闲聊/管理/划重点中三次出现同一个词，只有划重点那次是信号
5. **输出完整 ≠ 信号准确**：对照课件验证知识准确性，对照常识验证信号合理性
6. **知识内容不是自动等于低价值**：进入课件对照通道后，如果课件上该内容是大标题/独立条目，应升级为正范围信号
7. **正信号内部的强度差异**：老师用第一人称直接确认（"我就一定要考"）> 一般提醒（"稍微注意一下"）；唯一性信号（"只有这两个"）> 包含性信号（"这个也会涉及"）

---

## 十一、迭代改进循环

```
第 N 次使用
  → 人工判定项 → 人判 + 人判理由
  → 新人判样本 → 自动补入 few-shot 库
  → 第 N+1 次使用：同类模式自动分类，人工判定比例下降
```

**目标曲线**：
- 第 1 次：人工判定 ~10-15%
- 第 2 次：~5-8%
- 第 3 次及以后：< 5%

当某类模式连续 3 次不再触发人工判定，可将其从 few-shot 库移入硬规则。
