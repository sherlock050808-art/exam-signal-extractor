# 闭卷资料 MD 模板（v3.0）

## ⚠️ 硬性结构约束

本模板定义了 **6 个顶层 Section**，不允许增加、删除或重排：

```
§1 封面信息
§2 使用说明 + 闭卷核心策略
§3 题型分布（顶层 Section 不可删除；其下的子题型如选择/名解/简答/分析/计算/编程可根据实际考试情况增删）
§4 考试情报与分值分布
§5 按章节展开的知识内容   ← 主体，每个节点一个 subsection
§6 按题型速查索引
附录（G/M代码速查表、公式表等）
```

**禁止事项**：
- ❌ 禁止新增"考试地图"章节（考试地图是中间产物，不是最终资料的一部分）
- ❌ 禁止新增"真题高频考点验证"章节
- ❌ 禁止新增"知识点汇总表"章节
- ❌ 禁止把多个节点合并到一个 subsection
- ❌ 禁止在正文中混入"→ §X.X"交叉引用（仅速查索引可以使用）

## §5 按章节展开的知识内容（核心）

### 5.0 渲染规则（硬性约束）

#### 5.0.1 组织结构

```
## {章节名}  【第X章 PPT p.xx-xx】

  ← 章节级 teacher_signal 汇总（如有）

  ### {节点 name}  【PPT p.xx】【教材 第X章第X节】

    ← 节点级 teacher_signal（如有）
    ← 重点框 / 答案内容（按 exam_layer 裁剪）
    ← 易错框（如有 traps）
    ← 融合框（如有 fusions）
    ← 公式框 / 代码块（如有 计算/编程 题型）
```

#### 5.0.2 考试层级标签

每个节点标题后必须标注 exam_layer 和 question_types：

```markdown
### {节点 name}  【PPT p.xx】 🟢必考 [名解] [简答]
### {节点 name}  【PPT p.xx】 🟡高概率 [选择/判断]
### {节点 name}  【PPT p.xx】 🟠可能 [选择/判断]
### {节点 name}  【PPT p.xx】 ⚪排除
```

对应关系：
| exam_layer | 标签 | 颜色 |
|-----------|------|------|
| 必考层 | 🟢必考 | green |
| 高概率层 | 🟡高概率 | orange |
| 可能层 | 🟠可能 | light orange |
| 排除层 | ⚪排除 | gray |
| 待确认 | 🔵待确认 | blue |

#### 5.0.3 内容详略规则（硬性约束）

| exam_layer | 重点框内容 | 是否输出公式/例题/代码 | 是否使用 keybox |
|-----------|-----------|---------------------|----------------|
| 🟢必考层 | **完整答案，可默写** | 是 | ✅ 必须 |
| 🟡高概率层 | **采分点 + 简要展开** | 是 | ✅ 可选 |
| 🟠可能层 | **一句话识别**（3-5 条要点，无 box） | 否 | ❌ 不用 box，纯文字 |
| ⚪排除层 | **仅标记"已排除，不考"** | 否 | ❌ 不输出正文 |
| 🔵待确认 | 一句话 + ⚠️"考前确认" | 否 | ❌ 不用 box |

#### 5.0.4 Box 类型（仅 4 种，不允许发明新 box）

| Box 类型 | 用途 | 来源字段 | 颜色方案（LaTeX） |
|----------|-----|---------|-----------------|
| **课堂信号框** | 老师原话 + 录音时间戳 | `teacher_signal` / `recording` | purple |
| **重点框 keybox** | 核心知识点答案 | 根据 `ppt_source`/`textbook_source` 定位原文后裁剪 | yellow/orange |
| **易错框 warnbox** | 常见错误 + 区分方式 | `traps` | red |
| **公式框 formulabox** | 公式/计算/代码 | question_types 含 计算/编程 | blue |

**禁止**：signalbox、verifybox、fusebox、graybox 等不属于以上 4 种的 box。

#### 5.0.5 融合区处理

`fusions` 不创建独立 box。处理方式：
- 在涉及的每个节点末尾，用小字标注"⚠️ 融合考点：与【{with}】组合出 {question_form}"
- 只在最后一个被融合节点处详细展开融合内容

### 5.1 每个节点的渲染流程（管道）

```
输入：一个 Knowledge Node（已注入考试意图标签）

Step 1: 写标题行
  格式：### {name}  【PPT p.xx】【教材 第X章第X节】 {exam_layer标签} {question_type标签}

Step 2: 写课堂信号框（如果 teacher_signal 或 recording 非空）
  格式：
  > **📌 课堂信号**
  > "{teacher_signal 原文}"
  > — 录音 {timestamp}

Step 3: 写重点框（如果 exam_layer ≠ 排除层）
  🟢必考层：
  > **重点**
  > 1. {完整定义/原理}
  > 2. {分类/特点}
  > 3. {应用/注意事项}
  > ...
  
  🟡高概率层：
  > **采分点**
  > - {关键点1}
  > - {关键点2}
  > - {简要展开}
  
  🟠可能层（不用 box，纯文字）：
  - {一句话识别}
  - {关键词1}、{关键词2}

  ⚪排除层：不输出正文，仅标记"已排除"

Step 4: 写易错框（如果 traps 非空）
  格式：
  > **⚠️ 易错**
  > - ❌ 错误：{trap}
  > - ✅ 正确：{clarification}

Step 5: 写融合标注（如果 fusions 非空）
  格式：
  > **🔗 融合考点**：与 {with} 组合 → {question_form}

Step 6: 写公式框（如果 question_types 含"计算"）
  格式：
  > **公式**
  > - {公式名}：$$公式$$
  > - 变量说明：...

Step 7: 写代码块（如果 question_types 含"编程"）
  格式：带注释的完整代码示例
```

### 5.3 完整示例（模拟"闭环控制"节点）

```markdown
## 绪论  【第1章 PPT p.6-77】

> **📌 本章课堂信号**
> 绪论只考小题，但基本概念要掌握。老师特别强调：数控机床组成、开环/闭环/半闭环、控制轴数与联动轴数、定位精度/重复定位精度/脉冲当量、加工中心。

### 数控机床的组成与工作原理  【PPT p.23-30】 🟢必考 [简答]

> **📌 课堂信号**
> "机床里面，数控机床的组成里面，有几个组成部分对吧？数控装置、伺服系统，还有这个机械系统。那么你们在里面都知道哪些是其中的控制部分，哪些是执行部分，这是什么对吧？"

> **重点**
> 1. **控制介质/输入装置**：以指令形式记载加工信息。
> 2. **数控装置**：接受输入的加工信息，经运算处理向伺服系统发出脉冲，属于**控制部分**。
> 3. **伺服系统**：把数控装置的脉冲信号转换成机床运动部件的机械位移，属于**执行部分**。
> 4. **机械系统**：主轴部分、进给系统、刀库、ATC、APC等，属于**执行部分**。
>
> 数控系统工作过程：**输入 → 译码 → 数据处理 → 插补 → 伺服输出**。
>
> 【PPT p.23-30】

### 数控系统的发展  【PPT p.11-15】 🟠可能 [选择/判断]

- NC→CNC→PC数控→开放式CNC，五个发展阶段。
- 关键词：硬件数控、计算机数字控制、微处理器、开放式系统。
- 不深入，看一眼有印象即可。

### 按控制方式分类  【PPT p.31-37】 🟢必考 [简答] [分析]

> **📌 课堂信号**
> "另外一个重点就是按照控制方式分类，分为开环、闭环，还有半闭环。开环、闭环、半闭环到底是什么含义，是不是？往年的这是重要的一个考点。"

> **重点**
> - **开环控制**：无位置反馈，信号流程单向；精度取决于伺服和机械精度；结构简单、成本低；适用于精度要求不高的中小型机床。
> - **半闭环控制**：检测伺服电机轴或丝杠角位移；精度低于闭环但系统稳定、调试容易；应用广泛。
> - **闭环控制**：直接检测工作台直线位移（光栅、感应同步器）；理论上可消除全部传动误差；系统复杂、造价高；适用于大型和精密机床。
>
> 【PPT p.31-37】

> **⚠️ 易错**
> - ❌ 错误：半闭环检测工作台位置
> - ✅ 正确：半闭环检测电机轴或丝杠角位移，闭环才检测工作台实际位置
> - ❌ 错误：控制轴数 = 联动轴数
> - ✅ 正确：控制轴数是能控制的轴数，联动轴数是能同时协调控制的轴数
```

## 渲染优先级速查

渲染每个节点时，按以下优先级决定详略：

```
exam_layer=必考层  → 完整答案 + keybox + 公式/代码 + traps + fusions
exam_layer=高概率层 → 采分点 + 可选 keybox + 公式/代码 + traps
exam_layer=可能层   → 3-5 条要点（纯文字，不用 box）
exam_layer=排除层   → 仅标记"已排除"
exam_layer=待确认   → 一句话 + "⚠️考前确认"
```

## Box 使用决策树

```
有 teacher_signal? ─── YES → 课堂信号框 (purple)
有 exam_layer ≠ 排除? ─ YES → 重点框 keybox (yellow/orange)
有 traps? ──────────── YES → 易错框 warnbox (red)
有 计算/编程 题型? ── YES → 公式框 formulabox (blue) / 代码块
有 fusions? ───────── YES → 文末融合标注（不设独立 box）
```
