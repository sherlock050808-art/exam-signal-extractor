# 闭卷资料生成工作流（v3.0）

## 前置条件

已完成：
1. 建立 Knowledge Node 知识树（`knowledge-base/`）
2. 注入考试意图标签（`exam_layer`、`question_types`、`teacher_signal`、`traps`、`fusions`）
3. 确认原始资料可访问（PPT PDF、教材、旧版资料）

---

## 步骤 1：生成题型分布页

遍历所有节点，按 `question_types` 分类：

```
题型分布
├── 选择/判断题
│   └── 列出 question_types 包含"选择/判断"的节点
├── 名词解释
│   └── 列出 question_types 包含"名解"的节点
├── 简答题
│   └── 列出 question_types 包含"简答"的节点
├── 分析题
│   └── 列出 question_types 包含"分析"的节点
├── 计算题
│   └── 列出 question_types 包含"计算"的节点
└── 编程题
    └── 列出 question_types 包含"编程"的节点
```

排序规则：
1. `exam_layer` 优先级：必考层 > 高概率层 > 可能层 > 待确认
2. 同层级内按章节顺序排列
3. 排除层不列出

---

## 步骤 2：生成考试情报页

汇总所有 `teacher_signal` 和 `recording`：

```
考试情报
├── 老师明确强调
│   └── exam_layer=必考层 的 teacher_signal
├── 老师提到
│   └── exam_layer=高概率层 的 teacher_signal
├── 明确排除
│   └── exam_layer=排除层 的 teacher_signal
└── 待确认
    └── exam_layer=待确认 的 teacher_signal
```

---

## 步骤 3：按章节展开知识内容

### 3.1 章节组织

按 `chapter` → `section` → `name` 组织，保持课程原始结构。

### 3.2 每个节点的渲染流程

```
节点 name
├── 标题行
│   └── name + 【PPT p.xx】+ 【教材 p.xx】+ 【大纲 xxx】
│
├── 课堂信号框（如果 teacher_signal 或 recording 非空）
│   └── 老师原话 + 录音时间戳
│
├── 重点框（根据 exam_layer 决定详略）
│   ├── 必考层：完整答案（从原始资料定位原文编写）
│   ├── 高概率层：采分点 + 简要答案
│   ├── 可能层：一句话识别 + 关键词
│   └── 排除层：不输出正文，仅标记"已排除"
│
├── 易错框（如果 traps 非空）
│   └── 常见错误 + 区分方式
│
├── 融合区框（如果 fusions 非空）
│   └── 综合题组合 + 答题框架
│
└── 公式框/例题框（如果 question_types 包含计算/编程）
    └── 从原始资料定位公式和例题
```

### 3.3 内容详略规则

| exam_layer | 重点框内容 | 是否输出公式/例题 |
|-----------|-----------|----------------|
| 必考层 | 完整答案，可默写 | 是 |
| 高概率层 | 采分点 + 简要展开 | 是 |
| 可能层 | 一句话识别 | 否 |
| 排除层 | 仅标记"排除" | 否 |
| 待确认 | 一句话 + "考前确认" | 否 |

## 步骤 4：定位原始资料原文

### 4.1 PPT 定位

```python

# → 打开文件 → 翻到第 33 页 → 提取该页标题及正文
```

### 4.2 教材定位

```python
# → 打开教材 → 找到对应章节 → 提取定义/公式/例题
```

```python
# 如果节点 name 匹配旧版资料中的知识点标题
# → 提取旧版资料中的重点框/例题框/公式框内容
```

## 步骤 5：生成按题型速查索引

```
速查索引
├── 选择/判断题高频点
│   └── 必考层 + 高概率层 且 question_types 含"选择/判断"
├── 简答题高频点
│   └── 必考层 + 高概率层 且 question_types 含"简答"
├── 分析题高频点
│   └── 必考层 + 高概率层 且 question_types 含"分析"
├── 计算题高频点
│   └── question_types 含"计算"
└── 编程题高频点
    └── question_types 含"编程"
```

## 步骤 6：生成附录

### 6.1 公式速查表

收集所有 `question_types` 含"计算"的节点中的公式，按章节排列。

### 6.2 G/M 代码速查表

从编程相关节点和原始资料中提取常用 G/M 代码。

### 6.3 课堂信号汇总表

按章节汇总所有 `teacher_signal`，便于考前快速过一遍。
