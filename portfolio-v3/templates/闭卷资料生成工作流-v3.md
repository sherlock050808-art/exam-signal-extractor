# 闭卷资料生成工作流（v3.0）

## 前置条件

已完成：
1. 建立 Knowledge Node 知识树（`knowledge-base/`）
2. 注入考试意图标签（`exam_layer`、`question_types`、`teacher_signal`、`traps`、`fusions`）
3. 确认原始资料可访问（PPT PDF、教材、旧版资料）

---

## ⚠️ 步骤 0：读取 MD 模板（强制，不可跳过）

**在生成任何 MD 内容之前，必须先读取模板文件：**

```
Read: templates/闭卷资料MD模板-v3.md
```

模板定义了 6 个固定 Section 和所有渲染规则。生成 MD 时必须严格遵循，不允许自由发挥 Section 结构。

**如果不读模板就开始写 MD，一定会出现以下问题（历史教训）：**
- 把"考试地图"写成独立 Section（它是中间产物，不属于最终资料）
- 把"真题验证"写成独立 Section（应融入各节点的重点框/易错框）
- 可能层内容过长（应该只有 3-5 条要点，不用 box）
- 发明新的 Box 类型（只能有 4 种 Box）

---

## 步骤 1：生成题型分布页

遍历所有节点，按 `question_types` 分类：

```
题型分布
├── 选择/判断题
│   └── 列出 question_types 包含"选择/判断"的节点
├── 名词解释
│   └── 列出 question_types 包含"名解"的节点
├── 简答题
│   └── 列出 question_types 包含"简答"的节点
├── 分析题
│   └── 列出 question_types 包含"分析"的节点
├── 计算题
│   └── 列出 question_types 包含"计算"的节点
└── 编程题
    └── 列出 question_types 包含"编程"的节点
```

排序规则：
1. `exam_layer` 优先级：必考层 > 高概率层 > 可能层 > 待确认
2. 同层级内按章节顺序排列
3. 排除层不列出

## 步骤 2：生成考试情报页

汇总所有 `teacher_signal` 和 `recording`：

```
考试情报
├── 老师明确强调
│   └── exam_layer=必考层 的 teacher_signal
├── 老师提到
│   └── exam_layer=高概率层 的 teacher_signal
├── 明确排除
│   └── exam_layer=排除层 的 teacher_signal
└── 待确认
    └── exam_layer=待确认 的 teacher_signal
```

---

## 步骤 3：按章节展开知识内容

### 3.1 章节组织

按 `chapter` → `section` → `name` 组织，保持课程原始结构。

### 3.2 每个节点的渲染流程

```
节点 name
├── 标题行
│   └── name + 【PPT p.xx】+ 【教材 p.xx】+ 【大纲 xxx】
│
├── 课堂信号框（如果 teacher_signal 或 recording 非空）
│   └── 老师原话 + 录音时间戳
│
├── 重点框（根据 exam_layer 决定详略）
│   ├── 必考层：完整答案（从原始资料定位原文编写）
│   ├── 高概率层：采分点 + 简要答案
│   ├── 可能层：一句话识别 + 关键词
│   └── 排除层：不输出正文，仅标记"已排除"
│
├── 易错框（如果 traps 非空）
│   └── 常见错误 + 区分方式
│
├── 融合区框（如果 fusions 非空）
│   └── 综合题组合 + 答题框架
│
└── 公式框/例题框（如果 question_types 包含计算/编程）
    └── 从原始资料定位公式和例题
```

### 3.3 内容详略规则

| exam_layer | 重点框内容 | 是否输出公式/例题 |
|-----------|-----------|----------------|
| 必考层 | 完整答案，可默写 | 是 |
| 高概率层 | 采分点 + 简要展开 | 是 |
| 可能层 | 一句话识别 | 否 |
| 排除层 | 仅标记"排除" | 否 |
| 待确认 | 一句话 + "考前确认" | 否 |

## 步骤 7：输出文件

### 7.1 输出顺序（硬性约束）

```
步骤 6 完成后，按以下顺序生成文件：

1. 按《闭卷资料MD模板-v3.md》结构组装 Markdown
   → 输出：{课程名}复习指导-闭卷冲刺版.md
   → 作用：内容审查点，检查内容正确性和完整性
   → 强制：必须严格遵循模板的 6 个顶层 Section，禁止新增章节

2. 将 Markdown 机械转换为 LaTeX
   → 输出：{课程名}复习指导-闭卷冲刺版.tex
   → 强制：.tex 的结构 = .md 的结构（一对一映射，不重新组织）
   → 参考：数控技术复习指导-闭卷冲刺版.tex（80%+ 格式一致）

3. 编译 LaTeX → PDF
   → 输出：{课程名}复习指导-闭卷冲刺版.pdf
```

### 7.2 关键原则

- **md 管内容，tex 管排版**：md 决定结构，tex 只做机械转换
- **md 结构 = tex 结构**：不在转换过程中新增/删除/重排章节
- **先 md 后 tex**：md 是审查点，发现结构问题在 md 阶段修正，不要在 tex 里改结构
- **md 遵循模板**：生成 md 时严格对照 `闭卷资料MD模板-v3.md`，确保 6 个 Section 完整且无多余章节
