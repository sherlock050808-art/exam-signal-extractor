# 开卷闭卷考试助手 Portfolio（v3.0）

这是个人考试复习方法的 Portfolio，从 v1.0《信息蒸馏 SOP》到 v2.0《从课堂录音中提取考试信号》，再到 v3.0《基于 Knowledge Node 的考试资料生成》。

## 定位

- **不是题库**：不记忆具体题目，而是沉淀一套可复用的考试信号提取与资料生成工作流。
- **半自动**：提供结构化步骤、Prompt 模板和检查清单，由 Agent 或自己分步执行。
- **开卷/闭卷双策略**：两种考试形式共用同一套七层考试地图，只在覆盖阈值和投入权重上有差异。
- **v3.0 新定位**：用 Knowledge Node 承载课程结构，用考试意图标签驱动输出，节点本身不存储完整内容，而是作为原始资料的查询入口。

## v3.0 与 v2.0 的核心区别

| 维度 | v2.0 | v3.0 |
|------|------|------|
| 输入处理 | 录音、课件、大纲混合处理 | 输入先分为**考试意图**和**考试内容**，两条线并行 |
| 内容组织 | 人工整理成文本资料 | 建立 Knowledge Node 知识树，节点直接来自 PPT/教材/大纲标题 |
| 节点作用 | 无 | 节点是**查询入口 + 意图标签载体**，不是内容容器 |
| 重要性表达 | 直接分类（必考/高概率/可能/排除） | 保持直接分类，**不量化概率** |
| 内容来源 | 人工编写 | 根据节点来源定位到原始资料（PPT/教材/旧版资料）原文 |
| 输出模板 | 闭卷/开卷双策略 | **直接复用 v2.0 模板**，只改变内容填充方式 |

## 三个 SOP 版本的关系

- **`从教学资料中提取考试信号-v3.0.md`**：v3.0 核心方法论。输入分意图/内容两条线，节点来自 PPT/教材/大纲结构，输出模板复用 v2.0。推荐用于需要建立可持续维护知识库的场景。
- **`从课堂录音中提取考试信号-完整方法论.md`**：v2.0 核心方法论，重点在六步信号提取和七层考试地图。适合单次考试快速整理。
- **`信息蒸馏SOP-考试复习.md`**：v1.0 原版，作为历史版本保留。

## v3.0 核心原则

1. **输入先分类**：考试意图和考试内容分开处理。
2. **意图线用旧版**：录音/大纲/真题决定考什么、重不重要，直接分类，**不量化概率**。
3. **内容线用新版**：从 PPT/教材/大纲的已有结构中提取 Knowledge Node，不人工切分。
4. **节点是查询入口**：节点存放定位和意图标签，正文内容在生成时根据来源从原始资料拉取。
5. **输出用旧版模板**：闭卷/开卷双策略直接复用 v2.0 已验证结构。
6. **排除层 = 必考层**：知道什么不考和知道什么会考同样重要。
7. **复盘是改进来源**：每次考试后必须填写考后复盘清单，沉淀为 few-shot 样本。

## 维护指南

- 每次考试后把新的复盘写入 `examples/`。
- v3.0 改进项先在 `references/从教学资料中提取考试信号-v3.0.md`、`SKILL.md` 和新模板中验证。
- v2.0 及更早版本的核心文件（`从课堂录音中提取考试信号-完整方法论.md`、`信息蒸馏SOP-考试复习.md`）保留不变，作为历史参考。
